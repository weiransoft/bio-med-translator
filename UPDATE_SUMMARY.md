## 🎯 更新目标
增强 bio-med-translator 的文件处理能力，预装 Openskills，实现自动识别和读取各种格式的文档（Word、PDF、Markdown、Excel 等）。

## ✨ 新增功能

### 1. 预装 Openskills 支持

#### 1.1 docx skill - Word 文档处理
- **功能**: 读取和解析 Microsoft Word (.docx) 文档
- **自动调用**: 检测到 .docx 文件时自动使用
- **能力**:
  - 提取全部文本内容
  - 解析表格结构和数据
  - 保持文档层级结构
  - 支持中文和英文文档

#### 1.2 pdf skill - PDF 文档处理
- **功能**: 读取和解析 PDF 文档
- **自动调用**: 检测到 .pdf 文件时自动使用
- **能力**:
  - 提取文本内容
  - 识别文档结构
  - 处理图表和公式
  - 支持加密 PDF（需密码）

#### 1.3 markdown skill - Markdown 文档处理
- **功能**: 读取和解析 Markdown 文件
- **原生支持**: 直接读取，无需额外 skill
- **能力**:
  - 完整 Markdown 语法支持
  - 保持格式和结构
  - 支持代码块和数学公式

#### 1.4 excel skill - Excel 表格处理
- **功能**: 读取和解析 Excel 文件 (.xlsx, .xls)
- **自动调用**: 检测到 .xlsx/.xls 文件时自动使用
- **能力**:
  - 读取多个工作表
  - 提取单元格数据
  - 保持表格结构
  - 支持公式计算结果

### 2. 智能文件识别和处理

#### 2.1 自动识别流程
```
用户提供文件路径 → 识别文件扩展名 → 自动调用对应 skill
  ↓
  - .docx / .doc → docx skill
  - .pdf → pdf skill
  - .md / .markdown → markdown skill
  - .xlsx / .xls → excel skill
  - .txt → 直接读取
  ↓
提取文本内容 → 执行翻译工作流 → 输出译文
```

#### 2.2 批量文件处理
- 支持同时处理多个文件
- 自动建立统一术语表
- 跨文件术语一致性检查
- 进度跟踪和状态显示

### 3. 文件读取最佳实践

1. **自动识别**: 根据文件扩展名自动选择读取工具
2. **错误处理**: 读取失败时提供清晰的错误信息和建议
3. **大文件处理**: 分块读取大型文档，避免内存溢出
4. **格式保持**: 尽可能保持原文档格式和结构
5. **多文件支持**: 支持批量处理多个文件

## 🔧 技术实现

### 1. Skill 调用机制

bio-med-translator 通过以下方式调用其他 skills：

```markdown
【文件识别】检测到 .docx 文件，调用 docx skill 读取

【文档分析】
- 文件名：clinical-trial-protocol.docx
- 文档类型：临床试验方案 (Clinical Trial Protocol)
- 领域识别：临床医学 + 抗体药物
- 加载术语库：core-terms.md + clinical-medicine.md + antibody-drugs.md
- 文档结构：15 页，包含 8 个章节，3 个表格

【翻译结果】
[完整翻译的文档内容，保持原有格式和结构]
```

### 2. 文件类型识别

使用文件扩展名进行快速识别：

```javascript
// 伪代码示例
function identifyFileType(filePath) {
  const extension = filePath.split('.').pop().toLowerCase();
  
  switch(extension) {
    case 'docx':
    case 'doc':
      return 'docx';
    case 'pdf':
      return 'pdf';
    case 'md':
    case 'markdown':
      return 'markdown';
    case 'xlsx':
    case 'xls':
      return 'excel';
    default:
      return 'text';
  }
}
```

### 3. 错误处理

提供友好的错误信息：

```markdown
【错误】无法读取文件

可能的原因：
1. 文件路径不正确
2. 文件格式不受支持
3. 文件已损坏或加密
4. 缺少必要的 skill

建议操作：
1. 检查文件路径是否正确
2. 确认文件扩展名
3. 尝试用其他程序打开文件
4. 运行 ./check-installation.sh 验证安装
```

## 📊 使用场景

### 场景 1: 药品注册申报资料翻译

**文件类型**: Word 文档 (.docx)  
**内容**: IND/NDA/BLA 申报资料  
**处理流程**:
1. 调用 docx skill 读取
2. 识别为药品注册文档
3. 加载核心术语库 + 临床医学术语库
4. 执行专业翻译
5. 保持文档格式和结构
6. 提供术语对照表

### 场景 2: 学术论文翻译

**文件类型**: PDF 文档 (.pdf)  
**内容**: 研究论文、综述文章  
**处理流程**:
1. 调用 pdf skill 读取
2. 识别为学术论文
3. 加载对应领域术语库
4. 保持学术格式和引用
5. 处理图表和公式
6. 提供专业术语解释

### 场景 3: 临床试验文档批量翻译

**文件类型**: 多个 Word 和 PDF 文档  
**内容**: 试验方案、知情同意书、统计分析计划  
**处理流程**:
1. 批量处理所有文件
2. 每个文件调用对应 skill
3. 建立统一术语表
4. 跨文件术语一致性检查
5. 提供文件翻译清单
